# 飞书授权助手 (`feishu_auth_helper.py`) 使用指南

该脚本用于通过 **OAuth 2.0 流程** 获取您的飞书“个人身份授权”。完成授权后，您的上传脚本即可代表您本人进行文件夹操作，绕过繁琐的机器人权限共享步骤。

## 1. 前置准备

在运行脚本前，请确保您的飞书自建应用已完成以下配置：

1.  **基础信息**：在飞书开放平台获取 `App ID` 和 `App Secret`，并填入 `feishu_secrets.json`。
2.  **机器人能力**：在“应用能力”中开启 **“机器人”** 能力并发布版本。
3.  **权限管理**：至少申请并开通以下权限，并**发布版本**：
    *   `drive:drive` (查看、编辑、导出云空间所有文件)
    *   `drive:file` (查看、编辑、下载、打印云空间所有文件)
    *   `drive:file:upload` (上传文件)
4.  **安全设置 (关键)**：
    *   在“重定向 URL”中添加一行：`https://open.feishu.cn`

## 2. 操作步骤

1.  **启动脚本**：
    在终端/控制台运行：
    ```bash
    python feishu_auth_helper.py
    ```
2.  **浏览器授权**：
    脚本会自动生成一个授权链接并尝试打开浏览器。如果未自动打开，请手动复制控制台中显示的 URL。
3.  **获取授权码 (Code)**：
    在浏览器点击“允许”后，页面会跳转到一个以 `https://open.feishu.cn` 开头的空白页。请查看浏览器地址栏，复制 `code=` 后面的一串字符。
4.  **贴回 Code**：
    回到终端，在提示 `Enter code:` 处粘贴该字符并按 **回车**。

## 3. 运行结果

授权成功后，脚本会自动执行以下操作：
- 向飞书换取 `user_access_token` 和 `refresh_token`。
- 将这些令牌自动保存到您的 `feishu_secrets.json` 文件中。
- 此时 `feishu_uploader.py` 将自动识别并切换到“个人身份模式”。

## 4. 常见问题 (FAQ)

*   **Q: 每次上传都要运行这个助手吗？**
    *   **A: 不需要。** 脚本会自动利用 `refresh_token` 在后台静默续期。只要您每个月至少运行一次上传任务，授权就永远有效。
*   **Q: 提示 `redirect_uri 违法`？**
    *   **A:** 请确认在“安全设置”中填写的重定向 URL 完全匹配 `https://open.feishu.cn`。
*   **Q: 什么时候需要重新运行？**
    *   **A:** 当上传脚本提示“自动刷新 Token 失败”时，通常是由于超长时间未使用导致授权彻底失效，此时需要重新运行本助手。
