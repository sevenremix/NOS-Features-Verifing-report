# 项目脚本文件功能及关联分析

本文档对项目根目录下的辅助脚本进行了梳理，明确了它们在流水线中的地位以及删除后的影响。

## 1. 核心流程脚本 (不可删除)

这些脚本被 `main.py` 直接调用，是自动化流水线的核心组成部分。

| 文件名 | 在流程中的位置 | 功能描述 | 删除影响 |
| :--- | :--- | :--- | :--- |
| **`Transform_simple_copy.py`** | **第一步** | 预处理原始 Feature 表，生成归一化的中间文件。 | **严重报错**。`main.py` 启动即失败，后续流程无法进行。 |
| `check_consistency.py` | 第二步 | 校验预处理后的数据一致性。 | `main.py` 报错，失去数据质量把控。 |
| `Feature_Case_Merging.py` | 第三步 | 关联测试报告，生成最终的合并 Excel。 | `main.py` 报错，无法获得最终结果。 |

## 2. 辅助与调试工具 (建议备份后清理)

这些脚本**未被 `main.py` 调用**，主要用于开发阶段的调试、数据探测或手动结果校验。

| 文件名 | 功能描述 | 建议处理方式 |
| :--- | :--- | :--- |
| `verify_output.py` | **手动校验工具**。用于检查最终生成的 Excel 格式、表头、对齐等是否符合预期。 | 建议保留，以便在修改渲染逻辑后进行自动化检查。 |
| `robust_excel_reader.py` | 鲁棒读取逻辑的原型。该核心逻辑已整合进 `Feature_Case_Merging.py`。 | 可删除。 |
| `test_read.py` | 临时的读取功能测试脚本。 | 可删除。 |
| `dump_headers.py` | 快捷打印 Excel 所有 Sheet 表头的工具。 | 可分入 `tools` 文件夹。 |
| `inspect_excel.py` | 查看 Excel 内部结构和原始数据的工具。 | 可分入 `tools` 文件夹。 |
| `inspect_report.py` | 针对测试报告内容的特定探测脚本。 | 可分入 `tools` 文件夹。 |
| `remove_column.py` | **数据清理工具**。专门用于从原始 Feature 表中彻底删除 `UT NOS` 列（具破坏性）。 | 可删除，通常用于预处理前的原始数据手动修正。 |
| `debug_strange_values.py` | **调试探测工具**。使用鲁棒读取逻辑专门搜索和测试特定复杂功能点的匹配情况。 | 可删除。 |
| `feishu_uploader.py` | **云端上传工具**。专门用于将 Excel 报告上传至飞书云文档，提供外部调用接口。支持机器人和个人身份双模式。 | **保留**。用于自动化交付。 |
| `feishu_auth_helper.py` | **授权工具**。用于手动获取飞书用户授权（OAuth2），生成 `user_access_token`。 | **关键工具**。配置身份时使用。 |
| `test_feishu_upload.py` | **测试脚本**。验证文件上传及更名功能是否正常。 | 测试后可删除。 |
| `feishu_auth_helper_guide.md` | **使用说明**。详细描述了如何进行飞书 OAuth2 授权操作的指南。 | **保留**。参考文档。 |

## 3. 临时与调试数据文件 (JSON)

这些文件是执行某些读取或探测脚本后自动生成的中间产物。

| 文件名 | 功能描述 | 建议处理方式 |
| :--- | :--- | :--- |
| `head_dump.json` | 导出的 Excel 表头结构信息（可能由早期调试脚本生成）。 | 可安全删除。 |
| `raw_rows.json` | 解析出的原始 Excel 数据行，通常用于验证读取逻辑的准确性。 | 可安全删除。 |
| `feishu_secrets.json` | **配置文件**。存储飞书 API 的 `app_id` 和 `app_secret`。 | **关键配置**。需正确填入密钥后才能使用上传功能。 |

## 4. 总结

*   **`main.py` 执行依赖**: 仅依赖 `Transform_simple_copy.py`, `check_consistency.py`, `Feature_Case_Merging.py`。
*   **清理建议**: 如果您希望保持根目录整洁，建议创建一个 `debug_tools` 文件夹，将第 2 部分中提到的脚本全部移动进去，而不是直接删除。这样在未来数据格式发生变化需要排查问题时，依然可以使用这些成熟的调试工具。
